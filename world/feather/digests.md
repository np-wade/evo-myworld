# Feather digest — evo agents and subagent protocol

**verifier.md** (158 lines) — evo's static audit subagent: pre-phase checks for training-data leakage, benchmark-command sanity, gate coverage, hypothesis specificity, resource compliance (~30s); post-phase checks for duration sanity, artifact reality, score reproducibility spot-check, gate compliance, hypothesis-vs-outcome alignment. Returns structured JSON verdict (block/warn/note severity levels); gates `evo run` for pre-phase, advisory for post-phase. Flags five false-progress patterns: test-set ingestion, eval items in synthetic data, reverse-engineered verifier, instruct-model substitution, training-objective mismatch.

**ideator.md** (178 lines) — evo's experiment proposal generator. Three briefs: `failure_analysis` (scan last N discarded/failed experiments for shared failure clusters → meta-work proposals), `literature` (multi-source web/arXiv/HF/GitHub research, parallel scans across 6 source types, filter against already-tried hypotheses, write proposals to `.evo/ideator/proposals.jsonl`), `frontier_extrapolation` (find steepest score gradients on best path → propose scale/combine/refine variants). Returns JSON summary with titles and count; orchestrator reconciles parallel ideator runs.

**benchmark-reviewer.md** (196 lines) — evo's benchmark auditor in two modes. `mode=audit` (pre-flight): checks per-task instrumentation, eval-set leakage, goodhart gates, plumbing correctness, determinism; blocks on missing per-task traces or leakage. `mode=review-experiment` (post-commit): reads per-task traces and richer eval-runner logs, classifies failures into categories (truncated, wrong-format, wrong-answer, hallucination, refusal, language-drift, prompt-misread, eval-error, unknown), writes per-task annotations via `evo annotate`, returns failure breakdown + top pattern. Output is JSON with findings or summary depending on mode.

**subagent/SKILL.md** (417 lines) — protocol that evo optimization subagents follow.Four-field brief (objective, parent node, boundaries/anti-patterns, pointer traces) + iteration budget. Commands: `evo new --parent <id>`, `evo run`, `evo scratchpad`, `evo status`, `evo show/show diff/traces/annotations`, `evo gate list/check/add`, `evo discard/restore/annotate`, `evo telemetry feedback`. Workflow: load category skill, read scratchpad, study pointer traces, iterate (form hypothesis → create experiment → edit worktree → verifier pre-check → run → analyze → annotate → decide), append to `.evo/ideator/proposals.jsonl` if ideator, never modify benchmark/gate/framework code. Handles remote worktree commands via explicit `--exp-id`.
